## 📖 講義 1: 分割統治流の奥義 — マスター定理(15 XP)

分割統治の漸化式 **T(n) = a·T(n/b) + O(n^d)**(a 個の子問題、サイズ 1/b、結合コスト n^d)の解は:

| 条件 | 解 | 直感 |
|---|---|---|
| d > log_b a | T(n) = O(n^d) | 根(結合コスト)が支配 |
| d = log_b a | T(n) = O(n^d log n) | 全階層が同額を負担 |
| d < log_b a | T(n) = O(n^(log_b a)) | 葉(子問題の数)が支配 |

**即勝負**: マージソート T(n)=2T(n/2)+O(n) → a=2,b=2,d=1, log₂2=1=d → **O(n log n)** ✓
二分探索 T(n)=T(n/2)+O(1) → **O(log n)** ✓

**カラツバ法(分割統治の金字塔)**: n 桁同士の掛け算を素朴にやると 4 つの半分サイズの掛け算
T(n)=4T(n/2)+O(n)=O(n²)。だが代数の工夫で **3 回**に減らすと
T(n)=3T(n/2)+O(n)=O(n^log₂3)≈**O(n^1.585)**。
「子問題を 1 個減らす」だけで漸近的に勝つ —— 分割統治設計の発想を体に刻め。

## 📖 講義 2: 貪欲流 — 「証明付きの欲張り」(15 XP)

貪欲法 = 各ステップで局所的に最良の選択をし、二度と振り返らない。
**速いが、正しさは自明ではない**。証明の定番が**交換論法(exchange argument)**:

> 「最適解が貪欲解と違うなら、最適解の選択を貪欲の選択に**交換しても悪化しない**ことを示す。
> 交換を繰り返せば最適解は貪欲解に変形できる。ゆえに貪欲解も最適」

### 名作 1: 区間スケジューリング
n 個の講演から、重ならない最大数を選ぶ。**「終了時刻が最も早いものから選ぶ」**が最適。
(交換論法: 最適解の最初の講演を「最早終了」に交換しても、後ろの余地は広がりこそすれ狭まらない)

### 名作 2: ハフマン符号
出現頻度から最適な可変長符号(木)を作る: **頻度最小の 2 つを繰り返し併合**(World 3 の最小ヒープで O(n log n))。
ZIP・JPEG の中で今日も働いている圧縮の基礎理論だ。

### 名作 3: 最小全域木(MST)
全頂点を最小コストでつなぐ木。

- **Kruskal 法**: 辺を軽い順に見て、**閉路を作らなければ採用** — 閉路判定は Union-Find(隠しダンジョン 5 の住人が正規軍に昇格)。O(E log E)
- **Prim 法**: 1 頂点から始め、木と外を結ぶ最軽辺を最小ヒープで選び続ける(ダイクストラの兄弟)
- 正当性は**カット性質**: 「頂点集合をどう 2 つに割っても、その間を渡る最軽辺は必ずある MST に属する」

## 📖 講義 3: 流れの武術 — 最大フローと最小カット(15 XP)

有向グラフの各辺に容量。源 s から槽 t へ流せる最大量は?

**フォード=ファルカーソン法**: **残余グラフ**(残り容量+逆向きの「押し戻し」辺)上で
s→t の増加道を探しては流す、を増加道がなくなるまで繰り返す。
逆辺が「一度流した決定の取り消し」を可能にする —— 貪欲の失敗を流量で償う仕組みだ。

**最大フロー最小カット定理**: 最大フロー値 = s と t を分断する辺集合の最小容量。
「ネットワークの弱点(ボトルネック)」と「流せる限界」は同じものの裏表。

**応用の広さが本体**: 二部マッチング(学生と研究室の割当)、画像の領域分割、
野球リーグの優勝可能性判定 —— 「これフローで解けるじゃん」と気づく目が 6.046 の卒業証書だ。

## 📖 講義 4: 乱択流 — サイコロを武器にする(15 XP)

- **ラスベガス型**: 答えは常に正しく、実行時間が運次第(例: 乱択クイックソート — どんな入力でも**期待** O(n log n)。最悪入力を敵に選ばせない)
- **モンテカルロ型**: 時間は確定、答えがたまに間違う(例: ミラー–ラビン素数判定 — 誤り確率は反復で 4⁻ᵏ に。暗号鍵生成の現場で毎日使われている)

**乱択クイックセレクト**: ソートせずに「k 番目の値」を期待 O(n) で得る。
解析の主役は World 9 の**期待値の線形性** —— 数学の剣がアルゴリズムを斬る瞬間を見よ。

**普遍ハッシュ**: ハッシュ関数を族からランダムに選べば、どんな敵対的キー集合にも
期待 O(1) を保証できる。World 2 の「平均 O(1)」に理論的な土台が入った。

## 📖 講義 5: 近似流 — 正しく諦める技術(15 XP)

World 13 の神託「NP 完全なら厳密多項式解を諦めよ」の続き。
**近似アルゴリズム** = 多項式時間で「最適の α 倍以内」を**保証**する解法。

### 名作: 頂点被覆の 2-近似
辺を任意に選び**両端点を採用**、覆われた辺を消す、を繰り返すだけ。
証明: 選んだ辺たちは互いに端点を共有しない(マッチング)ので、最適解も各辺から最低 1 点は採る。
我々は 2 点ずつ採った → 高々 2 倍。∎ —— **最適解のサイズを知らずに比率を証明できる**のが鮮やかだ。

### 名作: 距離が三角不等式を満たす TSP の 2-近似
MST を作り(講義 2!)、その外周を歩き、重複頂点を飛ばす。
巡回路 ≦ MST の 2 倍 ≦ 最適巡回路の 2 倍。

**設計の流儀まとめ**: 厳密 DP / 分割統治 → 貪欲(証明できれば最速)→ 乱択 → 近似 → ヒューリスティック。
問題を見て**どの流派で構えるか**を選べることが、上級者の証である。

## ⚔️ クエスト

### クエスト 1: マスター定理の素振り(30 XP)

次の漸化式を解け。

1. T(n) = 4T(n/2) + O(n)
2. T(n) = 2T(n/2) + O(n²)
3. T(n) = 3T(n/3) + O(n)
4. カラツバ法が T(n) = 3T(n/2) + O(n) になる理由を、3 つの部分積の式とともに書け。

<details>
<summary>📜 答え</summary>

1. log₂4 = 2 > 1 → **O(n²)**(葉が支配)
2. d=2 > log₂2=1 → **O(n²)**(根が支配)
3. log₃3 = 1 = d → **O(n log n)**
4. x = a·2^(n/2) + b, y = c·2^(n/2) + d のとき xy に必要な ac, bd, (ad+bc) のうち、ad+bc = (a+b)(c+d) − ac − bd で求まるので掛け算は ac, bd, (a+b)(c+d) の 3 回で足りる。
</details>

### クエスト 2: 交換論法の修行(30 XP)

「終了時刻最早を選ぶ」区間スケジューリング貪欲の正当性を、交換論法で**自分の言葉で**証明せよ。
(「最適解の 1 番目を貪欲の 1 番目に置き換えても矛盾しない」→ 帰納的に全部置き換わる、の流れ)

### クエスト 3: 💻 実装クエスト「三流派の試合」(70 XP)

1. **Kruskal 法** を Union-Find ごと実装し、5 頂点程度のグラフで MST を求めよ
2. **乱択クイックセレクト**(k 番目の最小値、期待 O(n))を実装せよ
3. **頂点被覆 2-近似** を実装し、小さいグラフで総当たりの最適解と比べて比率を確認せよ

### クエスト 4: フローの目(30 XP)

1. 「n 人の応募者と m 件の仕事、各人は応募可能な仕事の集合を持つ。最大何組をマッチできるか」をフロー問題として定式化せよ(図で: 源・槽・容量)。
2. 最大フロー最小カット定理を使うと、マッチングの最大値が「ある集合のサイズ」で上から抑えられる。どんな集合か。

<details>
<summary>📜 答え</summary>

1. 源 s → 各応募者(容量 1)、応募者 → 応募可能な仕事(容量 1)、各仕事 → 槽 t(容量 1)。最大フロー値=最大マッチング数。
2. 最小カット。たとえば「一部の応募者と一部の仕事を選んで s/t を分断する辺集合」のサイズ。König の定理(二部グラフの最大マッチング=最小頂点被覆)がこの特殊形。
</details>

## 👹 ボス戦: 競技場の覇者「グランドマスター・アルゴ」(150 XP)

ノートを閉じて挑め。7 割以上で勝利。

1. マスター定理の 3 ケースを述べ、それぞれ「木のどこがコストを支配するか」の直感を添えよ。
2. 貪欲法を正当化する交換論法の骨格を 3 文で述べよ。
3. Kruskal 法のアルゴリズムと、閉路判定に使うデータ構造・その計算量を述べよ。
4. 残余グラフの「逆辺」は何のために存在するか。逆辺なしのナイーブ貪欲が失敗するグラフの例を描け。
5. ラスベガス型とモンテカルロ型の違いを、例とともに述べよ。
6. 頂点被覆 2-近似の証明を再現せよ。
7. 新しい問題に出会ったとき、厳密解法から近似・ヒューリスティックまでをどの順で検討するか、判断基準とともに述べよ。

<details>
<summary>📜 解答の骨子</summary>

1. 講義 1 の表。d > log_b a: 根の結合コスト支配 / 等しい: 各階層均等で log n 階層分 / d < log_b a: 葉の個数 a^(log_b n) = n^(log_b a) が支配。
2. 最適解 OPT が貪欲解 G と最初に食い違う選択を取り、OPT のその選択を G のものに交換しても解の品質が落ちないことを示す。これを繰り返すと OPT は G に変形できる。よって G は最適。
3. 辺を重み昇順にソートし、両端が異なる集合なら採用して union。閉路判定は Union-Find(経路圧縮+ランク統合)で 1 回ほぼ O(α(n)) ≈ 定数、全体 O(E log E)(ソート支配)。
4. 一度流した流れを部分的に取り消し、別の経路に振り直すため。例: s→a→t と s→b→t に各容量 1、a→b に容量 1 で、最初に s→a→b→t と流すと逆辺なしでは 1 で行き詰まるが、真の最大フローは 2。
5. ラスベガス: 出力は常に正しいが時間が確率変数(乱択クイックソート)。モンテカルロ: 時間は決定的だが誤り確率がある(ミラー–ラビン)。反復で誤り確率は指数的に下げられる。
6. 採用した辺集合はマッチング(端点素)。最適被覆は各辺につき少なくとも 1 端点を含むので |OPT| ≥ マッチングサイズ k。我々の解は 2k。よって 2k ≤ 2|OPT|。
7. ① 問題サイズと構造を確認(小さければ全探索・指数でも可)② DP/分割統治/貪欲(証明込み)で多項式厳密解を探す ③ NP 完全らしければ帰着で確認 ④ 近似比保証が要るなら近似アルゴリズム、要らなければ局所探索などのヒューリスティック ⑤ 特殊構造(木・平面・整数が小さい)があれば擬多項式 DP や専用解法。
</details>
