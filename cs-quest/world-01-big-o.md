## 📖 講義 1: なぜ「秒数」で測らないのか(10 XP)

あるプログラムが手元の PC で 3 秒かかったとしよう。これは性能の指標になるだろうか?
ならない。**マシンが変われば秒数は変わる**からだ。CS では、入力サイズ *n* が大きくなったとき
**操作回数がどう伸びるか**(成長率)でアルゴリズムを評価する。これを**漸近解析**と呼ぶ。

### Big-O の定義(ここだけは大学の数学)

関数 f(n) が **O(g(n))** であるとは:

> ある定数 c > 0 と n₀ が存在して、すべての n ≥ n₀ で f(n) ≤ c·g(n) が成り立つこと

直感で言えば「**n が十分大きいとき、f は g の定数倍を超えない**」。
つまり Big-O は**成長の上限**を表す。

- **O(g)** … 上限(これ以上悪くならない)
- **Ω(g)** … 下限(これ以上良くならない)
- **Θ(g)** … 上限かつ下限(ぴったりこの成長率)

### 計算のルール

1. **定数倍は無視**: 3n² + 5 → O(n²)
2. **低次の項は無視**: n² + n + 1 → O(n²)
3. **逐次実行は足し算**: O(n) のあとに O(m) → O(n + m)
4. **ネストしたループは掛け算**: n 回ループの中で n 回ループ → O(n²)

### 代表的な計算量クラス(山の標高一覧)

| 標高 | 記法 | 名前 | 代表例 | n = 1,000,000 のときの操作回数の目安 |
|---|---|---|---|---|
| 🟢 平地 | O(1) | 定数時間 | 配列の添字アクセス、ハッシュ表の検索(平均) | 1 |
| 🟢 丘 | O(log n) | 対数時間 | 二分探索 | 約 20 |
| 🟡 山道 | O(n) | 線形時間 | 配列の全走査 | 1,000,000 |
| 🟡 尾根 | O(n log n) | 線形対数 | マージソート、ヒープソート | 約 20,000,000 |
| 🟠 絶壁 | O(n²) | 二乗時間 | バブルソート、二重ループ | 10¹² (もう厳しい) |
| 🔴 雲の上 | O(2ⁿ) | 指数時間 | 部分集合の全列挙 | 宇宙の寿命でも終わらない |
| 💀 異次元 | O(n!) | 階乗時間 | 順列の全列挙 | 考えるだけ無駄 |

**覚え方**: ログは「半分にしていく」、n log n は「分割統治ソート」、2ⁿ は「選ぶ/選ばないの全パターン」。

### 最悪・平均・償却

- **最悪計算量**: 一番意地悪な入力での性能。面接で「計算量は?」と聞かれたら通常これ
- **平均計算量**: 入力がランダムだと仮定したときの期待値(例: クイックソートは平均 O(n log n))
- **償却計算量**: たまに高くつく操作をならした値(例: 動的配列の append は拡張時 O(n) だが、ならすと O(1))

### クエスト 1: オーダーの鑑定士(10 XP)

次のコードの時間計算量を答えよ。

```python
# (a)
def q1a(arr):
    total = 0
    for x in arr:
        total += x
    return total

# (b)
def q1b(arr):
    for i in range(len(arr)):
        for j in range(i + 1, len(arr)):
            print(arr[i], arr[j])

# (c)
def q1c(n):
    count = 0
    while n > 1:
        n = n // 2
        count += 1
    return count

# (d)
def q1d(arr):           # arr の長さ n
    arr.sort()          # 組み込みソート
    for x in arr:
        print(x)
```

<details>
<summary>📜 答え(自分で解いてから開け)</summary>

- (a) **O(n)** — 全要素を 1 回ずつ
- (b) **O(n²)** — ペアの数は n(n-1)/2 ≒ n²/2、定数倍は無視
- (c) **O(log n)** — n を半分にし続ける回数
- (d) **O(n log n)** — ソート O(n log n) + 走査 O(n)、大きい方が支配する
</details>

### クエスト 2: 嘘つき看板を見破れ(20 XP)

山道に立つ次の看板のうち、**間違っている**ものをすべて指摘し、理由を述べよ。

1. 「O(2n) は O(n) より遅い」
2. 「O(n²) のアルゴリズムは O(n log n) のアルゴリズムより常に遅い」
3. 「二分探索はソート済み配列に対して O(log n)」
4. 「f(n) = n² + 1000000n は O(n²)」

<details>
<summary>📜 答え</summary>

1. **嘘** — 定数倍は無視するので O(2n) = O(n)。同じ。
2. **嘘** — 漸近的には n² が遅いが、「常に」ではない。n が小さいときは定数倍次第で n² の方が速いこともある(実際、実用のソートは小さい区間で挿入ソートに切り替える)。
3. **正しい**
4. **正しい** — 低次の項は n がどれだけ係数が大きくても、いずれ n² に追い抜かれる。
</details>

### クエスト 3: 償却の理(20 XP)

空の動的配列(容量 1)に要素を n 個 append する。容量が一杯になったら容量を 2 倍にして全要素をコピーする。
コピー回数の合計が高々 2n 回以下であることを説明し、append 1 回あたりの償却計算量が O(1) であることを示せ。

<details>
<summary>📜 答え</summary>

容量拡張は容量が 1→2→4→…→n に達するまで起こり、各拡張でのコピー回数は 1, 2, 4, …, n/2。
合計は 1 + 2 + 4 + … + n/2 < n(等比数列の和)。挿入そのものが n 回なので総操作は n + n = 2n 以下。
n 回の操作で合計 O(n) なので、1 回あたりは O(n)/n = **O(1)(償却)**。
</details>

## 👹 ボス戦: 山頂の番人「オーダー・ドラゴン」(100 XP)

**ルール**: ノートを閉じ、制限時間 30 分。7 割(3.5 問)以上で勝利。

1. Big-O の定義を、定数 c と n₀ を使って正確に述べよ。
2. 次を成長率の遅い順(速く終わる順)に並べよ: `n!`, `n log n`, `2ⁿ`, `n`, `log n`, `n²`, `1`
3. 「このアルゴリズムは O(n²) である」と「このアルゴリズムは Θ(n²) である」の違いを説明せよ。
4. ループの中で `i = i * 3` と 3 倍にしながら n まで数えるループの計算量は?
5. ある操作列で、ほとんどの操作が O(1)、k 回に 1 回だけ O(k) かかる。償却計算量は?

<details>
<summary>📜 解答(戦いが終わってから開け)</summary>

1. ∃c>0, ∃n₀, ∀n≥n₀: f(n) ≤ c·g(n) のとき f(n) = O(g(n))。
2. `1` < `log n` < `n` < `n log n` < `n²` < `2ⁿ` < `n!`
3. O は上限のみ(実際はもっと速いかもしれない)。Θ は上限かつ下限(ぴったり n² で成長する)。
4. **O(log₃ n) = O(log n)** — 対数の底は定数倍の違いなので無視。
5. k 回の操作で合計 O(k) + O(k) = O(k) → 1 回あたり **O(1) 償却**。
</details>
