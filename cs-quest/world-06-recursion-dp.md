## 📖 講義 1: 砂丘の自己相似 — 再帰(10 XP)

**再帰**とは「問題を、同じ形の小さい問題で解く」こと。設計の掟は 2 つだけ:

1. **基底ケース(base case)**: これ以上小さくできない問題の答えを直接返す
2. **再帰ケース**: 自分より**確実に小さい**問題を呼び、結果を組み立てる

```python
def factorial(n):
    if n <= 1: return 1          # 基底ケース
    return n * factorial(n - 1)  # 再帰ケース(n が確実に減る)
```

**コールスタックを見よ**: 再帰呼び出しのたびにスタックフレーム(引数・戻り先)が積まれる。
深さ d の再帰は O(d) のメモリを食う。深すぎるとスタックオーバーフロー —— 砂に呑まれる。

**末尾再帰**: 再帰呼び出しが関数の最後の操作なら、ループに書き換えられる
(言語によっては自動最適化される。ならない言語も多いので、深い再帰はループ化を検討)。

## 📖 講義 2: 砂漠の罠 — 素朴な再帰の爆発(10 XP)

フィボナッチ数 F(n) = F(n−1) + F(n−2) を素朴に再帰すると:

```
                    F(5)
                  /      \
              F(4)        F(3)
             /    \      /    \
          F(3)   F(2)  F(2)  F(1)
          /  \
       F(2) F(1)   ← F(3) も F(2) も何度も解いている!
```

呼び出し回数は **O(2ⁿ)** — 指数爆発だ。原因は**部分問題の重複**。
同じ問いを何度も解いている。これが「砂漠で果てる」者の末路である。

## 📖 講義 3: 遺跡の知恵 その一 — メモ化(トップダウン DP)(10 XP)

**答えを一度刻んだら、二度と解かない。**

```python
memo = {}
def fib(n):
    if n <= 1: return n
    if n in memo: return memo[n]   # 刻まれた答えがあれば使う
    memo[n] = fib(n-1) + fib(n-2)  # なければ解いて刻む
    return memo[n]
```

部分問題は n + 1 個しかないので **O(n)** に激減。再帰の形を保ったまま高速化できるのが利点。

## 📖 講義 4: 遺跡の知恵 その二 — ボトムアップ DP(10 XP)

再帰をやめ、**小さい問題から順に表を埋める**。

```python
def fib(n):
    if n <= 1: return n
    dp = [0] * (n + 1)
    dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]
```

### DP が使える 2 条件

1. **部分問題の重複**: 同じ小問題が何度も現れる
2. **最適部分構造**: 大きい問題の最適解が、小さい問題の最適解から組み立てられる

### DP 設計の定石(遺跡の石板に刻まれた手順)

1. **状態を定義する**: 「dp[i] = ___ の答え」を日本語で言い切る(最重要!)
2. **遷移式を立てる**: dp[i] を、より小さい状態でどう表すか
3. **基底ケース**を決める
4. **埋める順序**を決める(依存する状態が先に埋まっているように)
5. 答えはどのマスか確認する

### 代表的な DP クエスト群(全冒険者が一度は通る道)

| 問題 | 状態の定義 | 遷移 |
|---|---|---|
| 階段の上り方(1 段 or 2 段) | dp[i] = i 段目までの上り方の数 | dp[i] = dp[i−1] + dp[i−2] |
| コイン両替(最小枚数) | dp[x] = 金額 x を作る最小枚数 | dp[x] = min(dp[x−c] + 1) for 各コイン c |
| 最長共通部分列 LCS | dp[i][j] = s の先頭 i 文字と t の先頭 j 文字の LCS 長 | 一致なら dp[i−1][j−1]+1、不一致なら max(dp[i−1][j], dp[i][j−1]) |
| 0/1 ナップサック | dp[i][w] = 品物 i 個まで・容量 w での最大価値 | 入れる / 入れないの max |

### クエスト 4: 石板の問答(20 XP)

1. メモ化とボトムアップ DP の長所・短所を比べよ。
2. 「貪欲法で解ける問題」と「DP が必要な問題」の違いは?
3. フィボナッチの DP のメモリを O(n) から O(1) に落とすには?

<details>
<summary>📜 答え</summary>

1. メモ化: 再帰のまま書けて直感的、必要な部分問題しか解かない。ただしスタック消費・関数呼び出しのオーバーヘッド。ボトムアップ: 速く省メモリにしやすいが、埋める順序を自分で設計する必要がある。
2. 貪欲法は「局所最適の選択が常に全体最適につながる」性質(貪欲選択性)が証明できるとき。それが成り立たず、選択の影響が後に波及する場合は DP で全選択肢を比較する。
3. dp[i] は直前 2 つにしか依存しないので、変数 2 個をローテーションすれば O(1)。
</details>

## 👹 ボス戦: 遺跡の守護神「スフィンクス」(100 XP)

ノートを閉じて挑め。7 割以上で勝利。スフィンクスは謎かけが好きだ。

1. 再帰関数が停止することを保証する 2 つの条件は?
2. 素朴な再帰フィボナッチが O(2ⁿ) になる理由と、メモ化で O(n) になる理由を「部分問題の個数」で説明せよ。
3. DP が適用できる 2 つの性質を述べよ。
4. 「dp[i][j] = 文字列 s の i 文字目までと t の j 文字目までの編集距離」と状態を定義した。遷移式を書け(挿入・削除・置換が各コスト 1)。
5. 0/1 ナップサックで「同じ品物を何個でも入れて良い」場合、遷移はどう変わる?
6. グリッドの左上から右下へ右・下移動のみで進む経路数を DP で定式化せよ。

<details>
<summary>📜 解答</summary>

1. 基底ケースが存在すること、再帰のたびに基底ケースへ確実に近づくこと。
2. 素朴版は同じ部分問題を重複して指数回解く。部分問題は実は n+1 個しかなく、メモ化すれば各々一度しか解かないので O(n)。
3. 部分問題の重複、最適部分構造。
4. `s[i] == t[j]` なら `dp[i][j] = dp[i−1][j−1]`。それ以外は `dp[i][j] = 1 + min(dp[i−1][j](削除), dp[i][j−1](挿入), dp[i−1][j−1](置換))`。
5. 「入れる」遷移で dp[i−1][w−wt] ではなく **dp[i][w−wt]**(同じ品物の行を再利用)を参照する。
6. dp[i][j] = マス (i,j) への経路数 = dp[i−1][j] + dp[i][j−1]、基底は最上行・最左列 = 1。
</details>
