## 📖 講義 1: 海図の読み方(10 XP)

**グラフ G = (V, E)** は頂点(vertex)の集合 V と辺(edge)の集合 E の組。

| 用語 | 意味 | 例 |
|---|---|---|
| 無向グラフ | 辺に向きがない | 友達関係 |
| 有向グラフ | 辺に向きがある | Twitter のフォロー、タスクの依存関係 |
| 重み付きグラフ | 辺にコストがある | 道路の距離、運賃 |
| 次数 | 頂点につながる辺の数 | 友達の人数 |
| パス / 閉路 | 頂点をたどる道 / 出発点に戻る道 | |
| 連結成分 | 互いに行き来できる頂点のかたまり | |
| DAG | 閉路のない有向グラフ | ビルド依存、履修の前提科目 |

### 2 つの表現方法

```
グラフ:  A — B
         |   |
         C — D
```

**隣接リスト**(实用ではほぼこちら):
```
A: [B, C]
B: [A, D]
C: [A, D]
D: [B, C]
```

**隣接行列**:
```
    A B C D
A [ 0 1 1 0 ]
B [ 1 0 0 1 ]
C [ 1 0 0 1 ]
D [ 0 1 1 0 ]
```

| | 隣接リスト | 隣接行列 |
|---|---|---|
| メモリ | O(V + E) | O(V²) |
| 「u→v の辺はあるか」 | O(次数) | O(1) |
| 隣接頂点の列挙 | O(次数) | O(V) |
| 向き不向き | 疎なグラフ(現実の大半) | 密なグラフ、辺の有無を頻繁に問う場合 |

## 📖 講義 2: 二人の航海士 — BFS と DFS(10 XP)

### ⚓ BFS(幅優先探索)— 「近い島から順に巡る」

**キュー**を使い、出発点から波紋のように広がる。

```
BFS(G, s):
    queue = [s], visited = {s}
    while queue が空でない:
        v = dequeue()
        for u in G.neighbors(v):
            if u not in visited:
                visited.add(u); enqueue(u)
```

- 計算量 **O(V + E)**
- **重みなしグラフの最短経路**(辺数最小)が求まる — これが BFS 最大の武器
- 訪問時に「親」を記録すれば経路復元もできる

### 🤿 DFS(深さ優先探索)— 「行ける所まで潜る」

**スタック**(または再帰)を使い、一本道を突き進んで行き止まりで戻る。

```
DFS(G, v):
    visited.add(v)
    for u in G.neighbors(v):
        if u not in visited:
            DFS(G, u)
```

- 計算量 **O(V + E)**
- 得意技: **閉路検出**、**連結成分の数え上げ**、**トポロジカルソート**、迷路の全探索

**使い分けの合言葉**: 「最短」と聞いたら BFS。「全部調べる・構造を調べる」なら DFS。

## 📖 講義 3: 依存関係の海図 — トポロジカルソート(10 XP)

**DAG** の頂点を「すべての辺が前→後ろを向く」ように一列に並べる。
履修順(微積 I → 微積 II)、ビルド順、タスクスケジューリングの数学的正体。

**DFS 法**: 各頂点の DFS が**完了した順**(帰りがけ順)を記録し、最後に逆順にする。O(V + E)。
**Kahn 法**: 入次数 0 の頂点をキューに入れ、取り出しては辺を消していく。閉路があると途中で詰まる(=閉路検出にもなる)。

## 📖 講義 4: 嵐の海の最短航路 — ダイクストラ法(10 XP)

辺に**非負の重み**があるとき、始点から全頂点への最短距離を求める。

**直感**: 「確定した港」から伸びる航路のうち、**最も安く着ける港を貪欲に確定**していく。
未確定の中の最小を高速に取り出すために **最小ヒープ(World 3 の隠者!)** を使う。

```
dijkstra(G, s):
    dist[v] = ∞ (全頂点), dist[s] = 0
    heap = [(0, s)]
    while heap が空でない:
        (d, v) = extract_min()
        if d > dist[v]: continue      # 古い情報はスキップ
        for (u, w) in G.neighbors(v):
            if dist[v] + w < dist[u]:
                dist[u] = dist[v] + w
                heap.insert((dist[u], u))
```

- 計算量 **O((V + E) log V)**(二分ヒープ使用時)
- ⚠️ **負の重みがあると壊れる**(その場合はベルマン・フォード法 — 隠しダンジョン参照)
- 地図アプリの経路探索の原型(実物は A* などで高速化 — これも隠しダンジョンに)

### クエスト 1: 海図の選択(20 XP)

各問題をグラフとしてモデル化せよ(頂点は何か、辺は何か、有向か、重みは何か、使うアルゴリズムは何か)。

1. 「友達の友達」を 2 ホップ以内で全部見つけたい
2. ソースコードのビルド順を決めたい(A が B を import している等)
3. 駅から駅への最短所要時間を求めたい
4. 迷路に出口があるか判定したい

<details>
<summary>📜 答え</summary>

1. 頂点=人、辺=友達関係(無向・重みなし)。**BFS** を深さ 2 まで。
2. 頂点=ファイル、辺=依存(有向)。**トポロジカルソート**。閉路があれば循環 import。
3. 頂点=駅、辺=路線(重み=所要時間、非負)。**ダイクストラ法**。
4. 頂点=マス、辺=隣接マス。**DFS でも BFS でも可**(到達可能性のみなら)。
</details>

### クエスト 4: 港の問答(20 XP)

1. BFS が重みなし最短経路を正しく求める理由を 1〜2 文で。
2. DFS で**有向**グラフの閉路を検出する方法は?(訪問状態を 3 色で管理)
3. ダイクストラ法に負の辺があるとなぜ壊れる?

<details>
<summary>📜 答え</summary>

1. BFS は距離 0, 1, 2, … の順に頂点を発見するので、最初に到達した時点の距離が最短。
2. 白(未訪問)・灰(訪問中=再帰スタック上)・黒(完了)で管理し、**灰色の頂点への辺(後退辺)を見つけたら閉路**。
3. ダイクストラは「一度確定した距離は二度と縮まない」前提の貪欲法。負の辺があると、確定後にもっと短い経路が現れうるため前提が崩れる。
</details>

## 👹 ボス戦: 大海の主「クラーケン」(100 XP)

ノートを閉じて挑め。7 割以上で勝利。

1. 隣接リストと隣接行列のメモリ・操作コストを比較し、それぞれが適する場面を述べよ。
2. BFS と DFS の擬似コードを書け(使うデータ構造の違いを明確に)。
3. V 個の島と E 本の航路。すべての島を訪れられるか(連結か)を O(V + E) で判定する方法は?
4. トポロジカルソートが存在する必要十分条件は?
5. ダイクストラ法の各ステップで「最小の未確定頂点を貪欲に確定してよい」のはなぜか(非負重みを使って説明せよ)。
6. 二部グラフ判定(頂点を 2 色に塗り分けられるか)を BFS でどう行う?

<details>
<summary>📜 解答</summary>

1. リスト: O(V+E)、隣接列挙が速い、疎グラフ向き。行列: O(V²)、辺の有無確認 O(1)、密グラフ向き。
2. BFS はキュー、DFS はスタック(または再帰)。骨格は講義 2 のとおり。
3. 任意の頂点から BFS/DFS して訪問数が V に等しいか確認。
4. グラフが **DAG(閉路なし有向グラフ)** であること。
5. 最小の未確定頂点 v より短い経路が後から見つかるには、未確定頂点を経由する必要があるが、その頂点までの距離はすでに v 以上であり、非負の辺を足しても縮まらないから。
6. BFS で訪問時に親と逆の色を塗る。同色同士を結ぶ辺を見つけたら二部グラフではない。
</details>
