## Forward Deployed Engineer とは何か？

Forward Deployed Engineer とは、**顧客の環境に直接入り込む**ソフトウェアエンジニアです。
オンサイトで、顧客のクラウド/VPC内で、顧客のワークフローの中に入り込み、自社のプラットフォームや
モデルを使って、その顧客固有の課題を解決する本番ソフトウェアを出荷します。

この職種は **Palantir**（FDEは「Deltas」と呼ばれます）で生み出され、AI業界全体に爆発的に広がりました。
OpenAI、Anthropic、Google、Cohere、Scale AI、Databricks、そして多くのスタートアップが、今やFDEを
採用し、フロンティアモデルをデプロイ済みで価値を生み出すシステムへと変えています。FDEの求人は2025年
1月から9月の間に **800%以上** 増加し、2026年にはOpenAIがデプロイメントに特化したジョイントベンチャー
「The Deployment Company」によって、このモデルを大規模に正式化しました。

**隣接する職種との一言での違い：**

| 職種 | 何に最適化するか |
|---|---|
| ソフトウェアエンジニア | 1つの能力を、**多数の**顧客が利用する |
| Forward Deployed Engineer | 多数の能力を、**1社の**顧客のために — その顧客の*本番環境*に出荷する |
| ソリューションアーキテクト | 助言、リファレンスアーキテクチャ、デモ（通常は本番コードを所有しない） |
| コンサルタント | レポートと提言（実装は他の誰かが行う） |
| MLエンジニア | モデルと学習パイプライン（通常は顧客と直接接しない） |

FDEは**実装を所有します**。顧客の本番システムで動く本物のコードを書き、それが機能するまで離れません。

### FDEが実際に行うこと（2025〜2026年の求人情報より）

- 戦略的顧客に常駐し、**フルスタックのLLMアプリケーション**、カスタムデータパイプライン、エージェント、
  MCPサーバー、評価ハーネスを設計・構築・デプロイする — 本番環境で。
- 重要度の高いデプロイメントの**主たる技術的オーナー**として振る舞う。本番が壊れたときの最後の砦となる。
- **ディスカバリー**を実施する。混沌としたビジネス課題を、構築可能で価値ある技術的ソリューションへと
  落とし込む。
- スタッフエンジニアからCISO、Cレベルの経営幹部まで、あらゆる相手とコミュニケーションし、プレッシャーの
  下でも技術的な一線を上品に守る。
- **再現可能なデプロイメントパターン**を見出し、プロダクトとエンジニアリングにフィードバックする。
- **曖昧さの中で自律的に動く** — 多くの場合、オンサイトでは自社から唯一のエンジニアとして。

### 企業が求めるもの（OpenAI / Anthropic / Palantir の求人を統合）

- 強力なプログラミングスキル — **Python** に加え、理想的には **TypeScript/JavaScript** または Java —
  本番アプリケーションを出荷した実績を伴うもの。
- **LLMの本番運用経験**：プロンプトエンジニアリング、エージェント、RAG、評価フレームワーク、
  大規模デプロイメント。
- フルスタックとデータの基礎：API、リレーショナルデータベース（Postgres/MySQL）、クラウド
  インフラ、コンテナ。
- エンタープライズの文脈：セキュリティ、プライバシー、コンプライアンス、顧客のVPC内での作業。
- 卓越したコミュニケーション能力：複雑な技術概念を非技術者向けに翻訳し、経営層にブリーフィングし、
  スコープを交渉する。
- 行動を重んじる姿勢。出張、そして他社の組織に常駐することへの抵抗のなさ。

### 報酬（米国、2025〜2026年の公開レンジ）

フロンティアラボのFDE職は、通常シニアプロダクトエンジニアと同等の報酬が支払われます。
OpenAI/Anthropic ではレベルと勤務地に応じておおよそ **総額20万〜42万ドル超** で、Palantir の FDSE
レンジはエントリーレベルではやや低めです。これらはあくまでスナップショットとして扱い、常に最新の
求人情報を確認してください。

## このカリキュラムの進め方

これは **約29週間のプログラム**です（パートタイムで週10〜15時間。フルタイムなら半分の期間）。少なくとも
1つの言語ですでにプログラミングができることを前提としています。各モジュールにはレッスン、演習、成果物、
チェックリストがあります — [Coding Interview University](https://github.com/jwasham/coding-interview-university) と同じように、進めながら項目を
チェックしていきましょう。

**黄金律：すべてのモジュールは成果物を生み出す。** FDEの採用はポートフォリオ重視です。修了する頃には、
デプロイ済みのLLMシステムからなる公開GitHubポートフォリオと、面接で実際に試される顧客対応スキルが
手に入ります。

- **約60%は構築** — プロジェクトこそがカリキュラムであり、読み物はそれを支えるものです。

  省略してもよさそうに感じますが、そうではありません。多くのエンジニアが選考プロセスで落ちるのは、
  まさにこの部分です。

## FAQ

**CSの学位は必要ですか？** いいえ。必要なのは、出荷できることを実証することです。このカリキュラムで
得られるポートフォリオに、しっかりしたCSの基礎（より深く学ぶには
[Coding Interview University](https://github.com/jwasham/coding-interview-university) を参照）を組み合わせるのが、現実的な道筋です。

**ML/研究のバックグラウンドは必要ですか？** いいえ。FDEはモデルを*使う*側であり、訓練する側では
ありません。必要なのは、モデルの*振る舞い*、コンテキストウィンドウ、ツール利用、そして評価を深く
理解することであって、バックプロパゲーションではありません。モジュール4でちょうど適切な深さを扱います。

**現役のSWEですが、先に進んでよいですか？** はい。フェーズ0の自己評価を受け、モジュール1〜2に目を通し、
時間はモジュール5〜10に費やしてください。あなたにおそらく欠けているのは、LLMアプリケーションの職人技と
顧客対応スキルです。

**「AIエンジニア」向けのカリキュラムとはどう違うのですか？** ざっくり言えば、AIエンジニア ⊂ FDE です。
FDEの役割は、LLMアプリケーションエンジニアリングの上に、エンタープライズデプロイメント、顧客への常駐、
ディスカバリー/スコーピング、経営層コミュニケーションを加えたものです。その違いがモジュール7〜9です。
