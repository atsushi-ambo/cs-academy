# モジュール0 — 前提知識とセットアップ（第1〜2週）

> 目標：動くソフトウェアを構築して出荷できることを確認し、ツールをセットアップし、プログラムの残りの
> 部分で埋めていくポートフォリオの土台を作る。

このカリキュラムはプログラミングをゼロから教える**ものではありません**。以下の自己評価に合格できない
場合は、まず2〜3か月を基礎に費やしてから（例：[Coding Interview University](https://github.com/jwasham/coding-interview-university)、CS50、
または *Python Crash Course*）、戻ってきてください。

---

## 自己評価 — 今日の時点で以下のすべてができるはず

- [ ] チュートリアルなしで約200行のPythonプログラム（CLIツール、スクレイパー、小さなAPIクライアント）を書ける。
- [ ] HTTPリクエストとは何かを説明できる：メソッド、ヘッダー、ステータスコード、JSONボディ。
- [ ] `JOIN`、`GROUP BY`、`WHERE` 句を含むSQLクエリを書ける。
- [ ] gitを使える：ブランチ、コミット、プッシュ、プルリクエストの作成、単純なマージコンフリクトの解決。
- [ ] スタックトレースを読み、print文やデバッガで自分のコードをデバッグできる。
- [ ] ネストしたループと、ハッシュマップの参照のBig-Oを説明できる。

5〜6個チェックできたら：先に進みましょう。3〜4個：進んでよいですが、モジュール2で余分な時間を確保して
ください。0〜2個：まず基礎を固めましょう。

## レッスン0.1 — あなたの2つの言語

FDEの求人は圧倒的に **Python**（LLMツールとデータ作業の言語）を求め、第2言語、通常は **TypeScript**
（Webフロントエンド、デモ、そしてMCPエコシステムの多くの言語）を強く好みます。このカリキュラムを通じて、
両方を使います。

- [ ] Python：仮想環境（`uv` または `venv`）、`pip`、型ヒント、`dataclasses`、`async`/`await` の基礎、
      `pytest` でのテスト記述に慣れている。
- [ ] TypeScript：`npm`/`pnpm`、基本的な型/インターフェース、`async`/`await` に慣れており、
      簡単なNodeスクリプトと簡単なReactページを作れる。

**リソース**
- *Python*：[公式チュートリアル](https://docs.python.org/3/tutorial/)、[uvドキュメント](https://docs.astral.sh/uv/)
- *TypeScript*：[TypeScript Handbook](https://www.typescriptlang.org/docs/handbook/intro.html)
- どちらかが初めての場合：最大1週間。言語はモジュールを構築しながら学ぶものであって、事前に学ぶものではありません。

## レッスン0.2 — ツールのセットアップ

- [ ] AIアシスタント付きのエディタ（VS Code / Cursor / Zed）。FDEには速さが求められます。AIコーディング
      ツールを使いこなすことは今や最低条件であり、ラボではそれ自体が仕事の一部です。
- [ ] 自分が使いこなせるターミナル：シェルの基本、`curl`、`jq`、SSH。
- [ ] Docker Desktop（または同等品）をインストールし、動作確認済みであること。
- [ ] アカウント＋APIキー：**Anthropic API**、**OpenAI API**、**Google AI Studio（Gemini）**。
      プログラム全体で20〜50ドルを見込んでください。ほとんどのモジュールはわずか数セントで済みます。
- [ ] 無料利用枠のあるクラウドアカウント（AWS、GCP、Azure のいずれか1つを選択。モジュール2でそこに
      デプロイします）。さらに、手早いデプロイ用に [Fly.io](https://fly.io)/[Railway](https://railway.app)/[Render](https://render.com)
      の無料アカウントも取得しておきましょう。
- [ ] GitHubプロフィールを整える：実名または一貫したハンドル、写真、自己紹介、ピン留めリポジトリの準備。
      この職種では、あなたのGitHubが**そのまま**履歴書になります。

## レッスン0.3 — ポートフォリオの土台作り

公開GitHubリポジトリ `fde-portfolio`（または個人サイト）を以下の構成で作成します。各モジュールの成果物は
すべてここからリンクされます：

```
fde-portfolio/
  README.md          <- プロジェクトの一覧。各項目に1段落＋デモリンク
  writeups/          <- 設計メモと意思決定ドキュメント（モジュール8でフォーマットを学ぶ）
  demos/             <- 録画デモへのリンク（Loom/YouTube限定公開）
```

- [ ] READMEの骨組みとともにリポジトリを作成。
- [ ] 「自分は何者で、何を目指して作っているか」を5行で書く。モジュール10で書き直します — それが狙いです。
      最初の草稿は残しておきましょう。

## レッスン0.4 — ベースラインプロジェクト（このプログラムの「hello world」）

週末で小さなエンドツーエンドのアプリを構築・デプロイします。これはあなたのツールチェーンが機能することを
証明し、自分の成長を測るための基準を与えてくれます。

**構築：「Ask My Docs」v0**
- テキストボックスのあるWebページ。ユーザーが質問を入力する。
- バックエンド（FastAPI または Express）が、その質問に加えて、自分で書いた2〜3個のMarkdownファイルの
  内容をプロンプトに詰め込み、1つのLLM APIを呼び出す。
- 答えを返す。RAGなし、ベクトルDBなし、ストリーミングなし。意図的に素朴に作ります — モジュール5でこれを
  きちんと作り直すことになり、その差分こそがあなたの学びです。

**成果物チェックリスト**
- [ ] 公開URLにデプロイ済み（Fly.io/Railway/Render）。
- [ ] GitHubにコードを置き、READMEを用意：何をするものか、実行方法、スクリーンショット。
- [ ] READMEに1段落：「この設計の何が問題か？」見つけられる弱点をすべて挙げる（コスト、レイテンシ、
      コンテキスト制限、評価なし、認証なし…）。保存しておくこと。モジュール6で比較します。
