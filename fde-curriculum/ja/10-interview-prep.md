# モジュール10 — 面接準備と就職活動（第29週〜）

> 目標: ポートフォリオをオファーに変える。FDEの選考ループはマルチモーダルだ — 実践的コーディング、
> システム設計、顧客シミュレーション、行動面接（behavioral）— そして各社がそれぞれに異なる重み
> づけをする。彼らがテストするスキルはすべて構築済みだ。このモジュールは、その見せ方と反復練習に
> ついてのものだ。

---

## レッスン10.1 — 選考ループを知る（モジュール1のリサーチを更新する — 求人とループは変化する）

**OpenAI（FDE/FDSE）** — 高速なループで、しばしば3〜4週間で約7回のタッチポイントがある: リクルーター
コール → 技術スクリーン → コーディングラウンド（実践的: 構築／統合／デバッグであって、アルゴリズムの
雑学ではない）→ かなりの分量の**テイクホーム（take-home、約5時間、彼らのAPIで本物を構築する）** →
システム設計 → **プレッシャー下での顧客への共感**を中心とした行動面接（「これを懐疑的なCISOに説明
してください」というシミュレーション — 場の空気を壊さずに技術的な一線を守る）→ リーダーシップ面談。

**Anthropic（FDE / Applied AI）** — リクルータースクリーン → 技術スクリーン → コーディング（実践的。
しばしばAIツールの使用を許可するポリシーがある — 確認し、どちらのモードでも優秀であれ）→ **顧客
会話シミュレーション**（あの有名なフィルター。モジュール8のディスカバリーの反復練習が対策になる）→
**eval 設計**に寄ったシステム設計（「それが機能していると、どうやって分かりますか？」）→ バリュー／
セーフティに関する会話（彼らが公開しているセーフティに関する見解を読み、暗唱ではなく本物の意見を
持つこと）。

**Palantir（FDSE）** — 分解（decomposition）を重視するコーディングスクリーン →「Palantir流」の
システム設計（雑然とした現実世界の制約、データ統合）→ ドライブ／オーナーシップのストーリーへの
強い重視。

**Google（Geminiに隣接したapplied AI／FDE的な職種）** — 古典的なGoogleのループに近い（ここでは
DSAが依然としてより重要 — モジュール3の取り組みが効いてくる）。加えてapplied AIのラウンドと
「Googleyness」の行動面接。

- [ ] ターゲット企業ごとに: 現行の求人を3件読み直し、最近の面接レポート（Glassdoor／Blind／Reddit）を
      検索し、1ページのブリーフを書く: ラウンド構成、重みづけ、自分のギャップ、自分の計画。

## レッスン10.2 — 4つのラウンドタイプと、あなたのドリル

### A. 実践的コーディング
- [ ] モジュール3の6つの実践ドリルを、何も見ずに、時間を計り、声に出して解説しながら再実行する。
- [ ] 新しいドリル: 不慣れなREST APIをドキュメントから45分で使いこなす。小さなFastAPIリポジトリに
      仕込まれたバグをデバッグする（友人かLLMに3つのバグを仕込んでもらう）。メモリに収まらない大きな
      ファイルをストリーム処理する。
- [ ] ラウンドでAIツールが許可されている場合: それらを*使って*練習する — 面接官は、あなたがAIの
      出力をどう指示し、検証し、修正するかを採点する。許可されていない場合: 使わずに練習する。
      両方のモードを身につけておく。

### B. システム設計（eval 中心を含む）
- [ ] モジュール3の5つの設計をゼロからやり直す — 今や2倍速く、より明快にできるはずだ。
- [ ] eval 設計ドリル（Anthropic流）:「顧客がRFPに自動で回答したがっている」のような5つのプロンプトに
      ついて、10分で次を作成する: 成功指標、ゴールデンセットの計画、グレーダー設計、デプロイメント
      ゲート、オンラインシグナル。プレイブックはモジュール6で書いた。それを反射になるまで圧縮する。
- [ ] 人間相手のフルモックを1回（tryexponent、interviewing.io、またはシニアの友人）。最初の本番ループ
      までに合計で最低2回のモック。

### C. 顧客シミュレーション
- [ ] モジュール8の押し戻し（pushback）シミュレーションを、新しいペルソナで再実行する: 不可能なSLAを
      求めるエグゼクティブ。CISO。「全部にファインチューニングすればいい」という熱狂者。引き出さなければ
      ならない沈黙のステークホルダー。録音し、レビューし、繰り返す。
- [ ] 普遍的な打ち手を、体で覚えるまでドリルする: **認める → 本当のニーズを明確化する → 共通の
      ゴールに再フレーミングする → 理由とともに実行可能な道筋を提示する → 指標を伴う次のステップに
      合意する。**
- [ ] あなたの「誠実なノー」の例を準備する: 顧客やステークホルダーが聞きたくないことを、親切に、
      代替案とともに伝えた経験。どのラボもこれを掘り下げてくる。

### D. 行動面接（Behavioral）
- [ ] 8〜10個のストーリーを、数字を伴うSTAR形式で書き、シグナルごとにインデックスを付ける: 曖昧さ、
      権限なきオーナーシップ、プレッシャー下での出荷、過度に広い要求のスコープダウン、解決した技術的
      対立、失敗とそこから変えたこと。あなたのキャップストーンが*そのまま*これらのストーリーだ — それ
      らをプロジェクトではなくエンゲージメントとして行ったのは、まさにそのためだった。
- [ ] 「なぜこの会社か／なぜFDEか」 — モジュール1で書いた段落を、これまで構築したすべてでアップ
      グレードする。特にAnthropic向けには: セーフティに本気で向き合う。OpenAI向けには: 出荷の速度と
      カスタマーオブセッション。Palantir向けには: オーナーシップとインパクト。

## レッスン10.3 — テイクホームのプレイブック

テイクホーム（take-home）は、機能の多さではなく、**1時間あたりのプロダクト判断力**で勝つ:
- [ ] 最初の30分をスコープに費やす: *コーディングの前に*READMEにミニSOWを書く（「スコープ内／
      スコープ外／なぜ」）。これらの企業のレビュアーは、これを即座に見抜く。
- [ ] 含めるもの: 小さな eval（10ケースでも — ほとんどの候補者はゼロだ）、APIパスのエラーハンドリング、
      コスト／レイテンシのメモ、そして「あと1週間あれば何をするか」のセクション。
- [ ] ワンコマンドで動くようにする。動かせなかったレビュアーは、それ相応の点数をつける。
- [ ] 1回練習する: 実際の過去のテイクホーム課題を選び（Resourcesのガイドに例がある）、5時間の
      厳格なタイムボックスで取り組み、その後、敵対的なシニアエンジニアのようにレビューする。

## レッスン10.4 — パイプラインの仕組み

- [ ] **レジュメ**: 1ページ、数字を伴う出荷物、上1/3 = あなたの最強のキャップストーンの指標。
      「エージェントシステムを構築・デプロイし、サポートチケットの38%をチケットあたり0.04ドルで自動
      解決した（eval スイート: 150ケース、精度94%）」 — この一文は、どんなスキルリストよりも多くを語る。
- [ ] **どこに応募するか**: 4つのラボ＋モジュール1のより広いリスト。検索すべきFDEに隣接した職種名:
      *Forward Deployed Engineer、Applied AI Engineer、Solutions Engineer (AI)、Deployment Engineer、
      Member of Technical Staff – Applied*。tier-2のリスト全体に幅広く応募する — 面接の練習は積み重なり、
      オファーはレバレッジを生む。
- [ ] **リファラルはポータルに勝る**: あなたのブログ記事＋ターゲット企業のFDEへの短く誠実なメモ
      （「Xを作り、その失敗モードをここに書きました。15分いただけませんか」）には、実際の成功率がある。
      これを10件やる。
- [ ] **順序立て**: 練習ティアのループを、ターゲットティアのループの2〜3週間前にスケジュールする。
- [ ] すべてをシートで追跡する: 会社、ステージ、日付、ラウンドごとの学び。各面接の後、記憶が新鮮な
      うちに質問を書き留め、48時間以内にギャップを埋める。

## レッスン10.5 — 最新であり続ける（この役割は速く動く）

採用されるまで（そしてされた後も）: 毎週 — 3つのラボすべてのリリースノート。毎月 — モジュール4の
モデル比較シートを更新し、1つのキャップストーンを新しいモデルに対して再テストする。*今月の*モデル
能力を流暢に語れるFDE候補者は、稀であり、記憶に残る。

## 最終チェックリスト —「応募する準備はできているか？」

- [ ] ポートフォリオ: TicketFlow AI＋3つのキャップストーン、すべて稼働、すべて eval の数字とデモ動画つき。
- [ ] 実践的コーディングドリルを、解説しながら、時間内に、パニックにならずにできる。
- [ ] eval ゲートを備えたLLMシステムを、ホワイトボードに35分で設計できる。
- [ ] 録音した顧客シミュレーションを3回以上やり抜き、それらをたじろがずに見返せる。
- [ ] 8〜10個の行動ストーリーを書き終え、「なぜFDEか」の答えが本物でリハーサル済み。
- [ ] ブログ記事を1本公開済み。アウトリーチのメモを10件送付済み。追跡シートが存在する。

これらの大半にチェックが付いたら: 応募しよう。100%準備ができたとは決して感じないものだ — FDEは
80%の確信で出荷し、反復する。あなたもそうすべきだ。頑張って — 取りに行こう。
